# Global OpenCode Preferences

## Package Managers

JavaScript/TypeScript projects use **bun** unless otherwise specified in the project AGENTS.md.

## Communication Style

- DO NOT use emojis
- DO NOT say "You're absolutely right" or similar validating phrases
- DO NOT say "I'm sorry" or apologetic phrases
- Be direct and concise

## Code Quality

- Prefer 100% test coverage when feasible
- Always run tests before considering work complete
- Use TDD approach when resolving bugs (write failing test first, then fix)
- ALWAYS use flexbox or grid for layouts, not margins (e.g. use `flex gap-*` instead of `space-y-*`, `mt-*`, `mx-auto`)

## Research

- Check official documentation before implementing
- Try `llms.txt` files first (e.g., `https://example.com/llms.txt`)
- Verify examples from docs before using

## Git

- DO NOT make git operations unless explicitly asked
- Use `/commit` command for commits (single-line, no co-authored-by)
